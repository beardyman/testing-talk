Testing in JS 101

What is a Test?

Code that executes and then checks other code

Reasons to Test

  • Stability
  • Confidence
  • Leads to better code
  • Does not guarantee bug free code

Types of Tests

  • Unit
  • Functional
  • Integration
  • Application
  • System

What Should I Test?

  • Everything
  • min / max values
  • border values

Four Phases of a Test

  • Setup
  • Execution of the SUT
  • Assertion of Results
  • Teardown

Example

System Under Test
          
function pow(number, power) {
  if (power === 1) {
    return number;
  }
  return number * pow(number, power - 1);
}
Test
            
it('should return the number passed for a power of 1', function () {
  var result = pow (5, 1);
  expect(result).to.equal(5);
});

it('should return the calculated power for a number', function () {
  var result = pow (5, 3);
  expect(result).to.equal(125);
});

Example

System Under Test
          
function pow(number, power) {
  if (power === 0) {
    return 1;
  } else if ( power < 0 ) {
    return pow(number, power + 1) / number;
  }
  return number * pow(number, power - 1);
}
Test
            
it('should return the calculated power for a negative number', function () {
  var result = pow (5, -3);
  expect(result).to.equal(0.008);
});

Testing Best Practices

  • Contains no logic
  • Idempotent
  • Independent
  • Clear and Concise
  • Treat Tests as First Class Code
  • Write Tests for how the code will be used

Code Coverage

Shows what code has be exercised and usually how many times.
